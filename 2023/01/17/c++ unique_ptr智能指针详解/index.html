C++ unique_ptr智能指针详解

本文详细介绍C++11引入的unique_ptr智能指针的特性和使用方法。

参考链接：https://blog.csdn.net/lemonxiaoxiao/article/details/108603916

基本概念

std::unique_ptr是C++11引入的智能指针，用于管理动态分配的对象。

核心特性

1. 独占所有权

unique_ptr不共享它的指针，同一时刻只有一个unique_ptr可以拥有对象的所有权。

2. 移动语义

unique_ptr只能通过移动语义转移所有权。

3. make_unique

std::make_unique是C++14引入的特性，提供更安全的方式创建unique_ptr。

4. 从函数返回

unique_ptr可以从函数中返回，所有权自动转移。

使用示例

使用注意事项

  1. 不要使用裸指针初始化多个unique_ptr
  2. 不要将unique_ptr传递给需要副本的函数
  3. 可以传递引用或使用std::move
  4. 优先使用make_unique而非new

unique_ptr vs shared_ptr

特性 unique_ptr shared_ptr
所有权 独占 共享
复制 不支持 支持
性能 更高（无引用计数） 稍低（有引用计数）
使用场景 独占资源 共享资源

总结

  1. unique_ptr是C++11引入的智能指针
  2. 采用独占所有权模式，不可复制
  3. 通过移动语义转移所有权
  4. make_unique是C++14特性，推荐使用
  5. 适用于独占资源的场景
