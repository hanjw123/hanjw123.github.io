单例模式与线程安全

单例模式是一种常用的设计模式，确保一个类只有一个实例。在多线程环境下，需要特别注意线程安全问题。保证线程安全的方法：加锁，为线程分配私有资源。本文介绍几种常见的单例实现方式及其线程安全性。

一、饿汉式单例

实现方式

特点

  • 线程安全：在程序启动时初始化，不存在竞争条件
  • 缺点：可能导致程序启动变慢，无法延迟初始化

二、懒汉式单例（非线程安全）

实现方式

特点

  • 非线程安全：多线程环境下可能创建多个实例
  • 优点：延迟初始化，按需创建

三、懒汉式单例（加锁版本）

实现方式

特点

  • 线程安全：使用互斥锁保护
  • 缺点：每次调用都需要加锁，性能开销较大

四、双重检查锁定（DCLP）

实现方式

特点

  • 线程安全：双重检查机制
  • 优点：只有首次调用需要加锁，性能较好

五、C++11局部静态变量方式

实现方式

特点

  • 线程安全：C++11标准保证局部静态变量初始化是线程安全的
  • 优点：简单、高效、线程安全
  • 推荐使用：这是C++11及以后版本的最佳实践

六、总结

实现方式 线程安全 延迟初始化 性能
饿汉式 是 否 高
懒汉式（无锁） 否 是 高
懒汉式（加锁） 是 是 低
DCLP 是 是 高
局部静态变量 是 是 高
