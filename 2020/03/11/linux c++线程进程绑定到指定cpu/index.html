Linux C++线程/进程绑定到指定CPU方法

本文介绍在Linux下使用C++将线程或进程绑定到指定CPU核心的方法。

参考链接：https://blog.csdn.net/fanyun_01/article/details/102787954

进程绑定到CPU

使用sched_setaffinity

线程绑定到CPU

使用pthread_setaffinity_np

使用std::thread（C++11及以上）

编译运行

使用场景

  1. 实时系统：确保关键线程在指定CPU上运行
  2. 性能优化：减少CPU缓存失效
  3. 多核编程：手动分配线程到不同核心
  4. 资源隔离：隔离不同类型的任务

注意事项

  1. CPU编号从0开始
  2. 绑定的CPU必须存在于系统中
  3. 需要适当的权限设置CPU亲和性
  4. 过度绑定可能导致负载不均衡

总结

在Linux下绑定线程/进程到指定CPU可以通过以下方法：

  1. 进程绑定：使用sched_setaffinity
  2. 线程绑定：使用pthread_setaffinity_np
  3. C++11线程：结合std::thread和pthread_setaffinity_np

合理使用CPU亲和性可以提高程序性能，特别是在多核系统上。
