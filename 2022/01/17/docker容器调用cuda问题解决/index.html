Docker容器调用CUDA问题解决方法

本文介绍Docker容器中调用CUDA时可能遇到的问题及解决方法。

参考链接：https://www.cnblogs.com/chester-cs/p/14444247.html

问题分析

问题1：容器内无法识别GPU

现象：在容器内运行nvidia-smi提示”Command not found”

原因：未添加--gpus参数，宿主机的GPU对容器不可见。

问题2：CUDA Version显示为N/A

现象：nvidia-smi输出中CUDA Version显示为N/A

原因：NVIDIA_DRIVER_CAPABILITIES环境变量设置不正确。

解决方案

正确的启动命令

参数说明

参数 说明
--gpus all 将所有GPU暴露给容器
NVIDIA_DRIVER_CAPABILITIES 指定驱动能力（compute:计算支持，utility:工具支持）
NVIDIA_VISIBLE_DEVICES 指定可见的GPU设备

NVIDIA_DRIVER_CAPABILITIES详解

  • utility：仅提供基础工具支持（如nvidia-smi）
  • compute：提供CUDA计算支持
  • graphics：提供图形支持
  • video：提供视频编解码支持
  • all：提供所有功能

验证方法

常见错误及解决

错误：docker: Error response from daemon: could not select device driver “” with capabilities: [[gpu]]

解决方案：

错误：Failed to initialize NVML: Driver/library version mismatch

解决方案：重启Docker服务或宿主机

最佳实践

  1. 使用官方NVIDIA CUDA镜像作为基础镜像
  2. 在Dockerfile中设置正确的环境变量
  3. 使用--gpus参数暴露GPU
  4. 正确设置NVIDIA_DRIVER_CAPABILITIES
  5. 在容器启动后验证GPU可用性
